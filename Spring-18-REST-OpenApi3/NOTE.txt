1. Resources to create Open API documentation

    Swagger documentation tool:
    https://app.swaggerhub.com/welcome
    https://editor.swagger.io/
    https://editor-next.swagger.io/

    Swagger documentation:
    https://github.com/OAI/OpenAPI-Specification/blob/main/versions/3.0.3.md#schema

    sample of generated Open API documentation is in openapi3_0.yaml file

 2. To create Open API for existing API project (Spring-16-REST-Jackson is taken as an example)

 2.1. Add pom.xml dependency

         <dependency>
             <groupId>org.springdoc</groupId>
             <artifactId>springdoc-openapi-ui</artifactId>
             <version>1.6.9</version>
         </dependency>

    after that when you run application OpenAPI generated automatically and available at:

    http://localhost:8080/v3/api-docs - to see autogenerated api config in xml format
        this location can be changed in application.property file with:
            springdoc.api-docs.path=/api-docs

    http://localhost:8080/api-docs.yaml - to see autogenerated api config in yaml format
    http://localhost:8080/swagger-ui/index.html
        this location can be changed in application.property file with:
            springdoc.swagger-ui.path=/swagger-custom.html

3. To be able to add api specific tags in api code you need to add openAPI bean in runner class.

        @Bean
        public OpenAPI customOpenApi() {
            return new OpenAPI()
                    .info(new Info().title("Cinema Application OpenAPI").version("v1").description("Cinema application API documentation"));
        }
